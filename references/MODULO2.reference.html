07 — Fórmulas Clave
Implementar exactamente — no aproximar
Horas de vida por transacción
horas = Math.round((amount / price_per_hour_snapshot) * 10) / 10
• Si price_per_hour_snapshot es null → no mostrar chip dorado
• price_per_hour_snapshot se guarda al crear la tx, nunca se recalcula
• Gastos → horas doradas negativas · Ingresos → horas verdes positivas
Neto + Tasa de retención
neto = totalIngresos - totalGastos
tasaRetencion = Math.round((neto / totalIngresos) * 100)
horasLibres = Math.round((neto / price_per_hour) * 10) / 10
• Si neto < 0 → hero label cambia a "déficit este mes", valor en rojo
• Si totalIngresos === 0 → tasaRetencion = 0, no dividir por cero
avg_amount — Promedio histórico de presupuestos
-- últimos 3 meses COMPLETOS (excluir mes actual)
SELECT category_id,
  SUM(amount) / COUNT(DISTINCT DATE_TRUNC('month', transaction_date)) AS avg_amount
FROM transactions
WHERE user_id = $1
  AND type = 'expense'
  AND transaction_date >= DATE_TRUNC('month', NOW()) - INTERVAL '3 months'
  AND transaction_date < DATE_TRUNC('month', NOW())
GROUP BY category_id
• Ejecutar en createTransaction() y deleteTransaction() — automático, nunca manual
• Upsert resultado en budgets con source = 'system'
• Si no hay historial → avg_amount = null, no mostrar barra de presupuesto
Plantilla recurrente pendiente
is_pending = is_active
  && day_of_month <= today.getDate()
  && !transactions.find(tx =>
    tx.recurring_template_id === template.id
    && tx.period_id === periodo_activo.id
  )
• Solo sugerir si el día ya pasó (day_of_month ≤ hoy)
• Al aprobar: createTransaction con recurring_template_id + status 'confirmed'
